# راهنمای استفاده بهبود یافته - Enhanced Usage Guide

## بهبودهای اصلی - Key Improvements

### ✨ 1. مرور و انتخاب دقیق مسیر - Accurate Path Selection

**چرا مهم است؟**
- جلوگیری از خطاهای تایپی مسیر
- تایید اینکه مسیر واقعا پروژه است
- دستیابی به اطلاعات محیط

**نحوه استفاده:**
1. در صفحه "افزودن پروژه" کلیک بر دکمه "مرور" کنید
2. از میانبرها یا مسیر مستقیم استفاده کنید
3. پوشه مورد نظر را انتخاب کنید
4. سیستم خودکار پروژه را تصدیق و دستورات را پر می کند

### ✨ 2. تشخیص خودکار نوع پروژه - Auto Project Type Detection

**پشتیبانی شده:**
```
Node.js    → npm start, npm stop
Python     → python app.py
Django     → python manage.py runserver
Docker     → docker-compose up, docker-compose down
Ruby Rails → bundle exec rails server
Go         → go run main.go
Rust       → cargo run
```

**فایده:**
- دستورات درست برای هر نوع پروژه
- پیشنهادهای متعدد دستور
- کاهش خطاهای دستوری

### ✨ 3. اعتبار سنجی کامل مسیر - Complete Path Validation

**بررسی های انجام شده:**
```
✓ آیا مسیر وجود دارد؟
✓ آیا یک پوشه است؟
✓ آیا قابل خواندگی است؟
✓ آیا پروژه معتبری است؟
✓ اطلاعات محیط (Node version, Python version, etc)
```

**مثال پاسخ:**
```json
{
  "valid": true,
  "projectType": "nodejs",
  "suggestions": {
    "startCommand": "npm start",
    "stopCommand": "npm stop"
  },
  "warnings": [
    "پوشه node_modules یافت نشد - node_modules not found"
  ]
}
```

### ✨ 4. متغیرهای محیطی - Environment Variables

**استفاده:**
```javascript
// Server-side: محیط متغیرها را دریافت می کند
const envVars = await db.getEnvironmentVariables(projectId);

// Client-side: اضافه کردن
POST /api/projects/:id/env
{
  "key": "DATABASE_URL",
  "value": "postgresql://user:pass@localhost",
  "isSecret": true
}
```

### ✨ 5. سیستم برچسب گذاری - Tagging System

**نمونه:**
```
پروژه: My App
برچسب ها: nodejs, api, production
```

**جستجو:** می توانید با برچسب جستجو کنید

### ✨ 6. خطاگیری بهتر - Better Error Handling

**پیام های واضح:**
```
❌ خطا: مسیر `/invalid/path` یافت نشد
💡 راهنما: مسیری که وارد کردید وجود ندارد، لطفا مرور را استفاده کنید
```

**جزئیات خطا:**
```
- تنها خطاهای موجود
- هشدارهایی که باید توجه کنید
- پیشنهادهای تصحیح
```

## رفع مشکلات - Troubleshooting

### مشکل: مسیر یافت نشد
```
❌ خطا: مسیر /invalid/path یافت نشد

✅ حل:
1. از دکمه مرور استفاده کنید
2. مسیر را دوبارة بررسی کنید
3. مسیر به صورت مطلق (absolute path) وارد کنید
```

### مشکل: دستور دستیافت نشد
```
❌ خطا: دستور npm پیدا نشد

✅ حل:
1. Node.js و npm را نصب کنید
2. مسیری صحیح وارد کنید
3. دستور دیگری را امتحان کنید
```

### مشکل: مسیر خالی است
```
⚠️ اخطار: پوشه خالی است

✅ حل:
1. پروژه خود را به این پوشه کپی کنید
2. پوشه پروژه درست است مطمئن شوید
```

## سوالات متداول - FAQ

**س: آیا می توانم مسیر را دستی وارد کنم؟**
ج: بله، می توانید مسیر مطلق (absolute path) را وارد کنید

**س: چگونه برچسب اضافه کنم؟**
ج: در فیلد "برچسب ها" برچسب هایی را وارد کنید که با کاما جدا شده باشند

**س: آیا می توانم محیط متغیرها را تغییر دهم؟**
ج: بله، به صفحه متغیرهای محیطی رفته و تغییرات لازم را انجام دهید

**س: دستورات خودکار چگونه تشخیص داده می شوند؟**
ج: سیستم فایل های مخصوص را جستجو می کند (package.json برای Node, requirements.txt برای Python)

**س: آیا اطلاعات حفاظت شده است؟**
ج: بله، متغیرهای حساس رمزگذاری شده ذخیره می شوند
